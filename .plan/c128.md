# Commodore 128 Machine Implementation Architecture & Plan

This document outlines the architecture and implementation roadmap for adding full Commodore 128 (C128) simulation support to `mmsim` (`mmemu`).

---

## 1. Architectural Overview

The Commodore 128 is a multi-mode, dual-CPU, dual-video system featuring 128 KB of RAM and backward hardware compatibility with the Commodore 64.

### Core Components
1. **MOS 8502 CPU**: An enhanced MOS 6510 variant capable of running at 1 MHz or 2 MHz (fast mode). Includes an on-chip 8-bit directional I/O port at `$00/$01`.
2. **Zilog Z80A CPU**: Secondary processor running at an effective 2 MHz (shared bus clock). Handles system startup and CP/M 3.0 execution.
3. **MOS 8722 MMU (Memory Management Unit)**: Controls physical RAM banking (128 KB split across two 64 KB banks), zero-page and stack relocation, RAM configuration blocks, shared RAM boundaries, and operating mode selection.
4. **MOS 8563 VDC (Video Display Controller)**: 80-column RGBI digital video display controller with dedicated video RAM (16 KB / 64 KB), 37 internal registers accessed via indirect index/data ports (`$D600`/`$D601`), and built-in block fill/copy hardware.
5. **MOS 8564/8566 VIC-IIe**: 40-column composite video chip extending the standard VIC-II with 2 MHz clock control (`$D030`) and VDC interface logic.
6. **Peripherals & I/O**: Dual MOS 6526 CIAs (`$DC00` and `$DD00`), MOS 6581/8580 SID (`$D400`), IEC serial bus interface, and expanded 108-key keyboard matrix.

### Operating Modes
- **C128 Mode**: 8502 CPU active, full 128 KB MMU banking, BASIC 7.0, VDC 80-column + VIC-IIe 40-column displays.
- **C64 Mode**: Hardware compatibility mode matching C64 PLA layout. Triggered by holding the Commodore key at boot or executing `GO 64`. Disables MMU extended functions and VDC.
- **CP/M Mode**: Z80 CPU active executing CP/M 3.0 OS using RAM Bank 0/1.

## 2. Implementation Roadmap

### Phase 34.1: MOS 8502 CPU & 2 MHz Fast Mode
*Goal: Implement the MOS 8502 CPU core with clock scaling support.*

- [ ] **`MOS8502` Core** (`src/plugins/devices/c128/cpu8502/cpu8502.h/cpp`):
  - Inherit from `MOS6510` (or `MOS6502`).
  - Implement 8-bit on-chip I/O port at `$00/$01` (Direction & Data registers).
  - Add 2 MHz speed mode state flag. When active, double CPU step/cycle execution during VIC-IIe blanking intervals.

### Phase 34.2: Zilog Z80A Core & Dual-CPU Switcher
*Goal: Add Z80 CPU core and dual-processor bus multiplexing.*

- [ ] **`Z80` Core** (`src/plugins/z80/cpu_z80.h/cpp`):
  - Implement `Z80` class inheriting from `ICore`.
  - Full register set: AF, BC, DE, HL, IX, IY, SP, PC, alternate register set (AF', BC', DE', HL'), I, R registers.
  - Implement Z80 opcode matrix, addressing modes, and interrupt modes (IM0, IM1, IM2).
  - Register descriptors and inline disassembler (`DisassemblerZ80`).
- [ ] **Dual-CPU Switcher** (`src/plugins/machines/c128/cpu_switcher.h/cpp`):
  - Multiplex bus access between 8502 and Z80 `ICore` slots based on MMU Mode Configuration Register (`$D505` bit 0).
  - Power-on sequence: Initialize with Z80 active; Z80 checks sector 0 of disk drive 8 / RAM, then yields control to 8502 by writing to `$D505`.

### Phase 34.3: MOS 8722 MMU Implementation
*Goal: Implement physical address translation, RAM banking, and shared RAM.*

- [ ] **`MMU8722` IOHandler** (`src/plugins/devices/mmu8722/mmu8722.h/cpp`):
  - Register mapping at `$D500–$D5FF`.
  - Configuration Registers CR0–CR4 (`$D500–$D504`).
  - Mode Configuration Register MCR (`$D505`): C128/C64 mode bit, Z80/8502 select bit, EXROM/GAME status.
  - RAM Configuration Register RCR (`$D506`): Selects RAM Bank 0/1 for 4 address blocks (`$0000–$3FFF`, `$4000–$7FFF`, `$8000–$BFFF`, `$C000–$FFFF`).
  - Page 0 / Page 1 Relocation Registers (`$D507–$D50A`): Relocate Zero Page and Stack to arbitrary physical RAM locations.
- [ ] **`C128MmuAdapter` Bus Wrapper**:
  - Intercept CPU reads/writes and translate 16-bit virtual addresses to 128 KB physical bus addresses across RAM Bank 0 and Bank 1.
  - Handle Shared RAM regions (`$0000–$00FF` and `$FF00–$FFFF`) which map to Bank 0 regardless of active bank setting.
  - Support ROM overlays: BASIC 7.0 (High/Low), KERNAL, Editor, CharROM, and Function ROMs (Internal/External).

### Phase 34.4: MOS 8563 VDC 80-Column Display Controller
*Goal: 80-column RGBI display controller with independent video RAM and block operations.*

- [ ] **`VDC8563` Core** (`src/plugins/devices/vdc8563/vdc8563.h/cpp`):
  - Implement `VDC8563` inheriting from `IOHandler` and `IVideoOutput`.
  - Indirect register interface at `$D600` (Address/Status) and `$D601` (Data).
  - Complete 37 internal register file (display timing, start address, cursor position/style, character attributes).
  - Dedicated 16 KB / 64 KB video RAM storage.
  - Hardware block copy and memory fill engine executed on register command.
- [ ] **VDC Rendering Pipeline**:
  - Render 80×25 text mode and monochrome/attribute graphic modes.
  - Process text attribute byte (blink, underline, reverse, highlight, RGBI foreground/background colors).
  - Generate RGBA pixel frame buffer for host GUI output.

### Phase 34.5: MOS 8564/8566 VIC-IIe Enhancements
*Goal: Extend standard VIC-II with C128 features.*

- [ ] **`VIC2e` Core** (`src/plugins/devices/vic2e/vic2e.h/cpp`):
  - Inherit from or wrap `VIC2` (Phase 11.4).
  - Add `$D030` Clock Control Register (bit 0 toggles 1 MHz / 2 MHz fast speed mode).
  - Implement 40/80 display key sensing and synchronization.

### Phase 34.6: C128 Machine Factory and Memory Map
*Goal: Assemble all components into a bootable C128 machine descriptor.*

- [ ] **`c128` Machine Descriptor** (`src/plugins/machines/c128/machine_c128.cpp`):
  - Create machine factory for `"c128"`.
  - Load ROM set: C128 KERNAL (32 KB), BASIC 7.0 (32 KB), Editor (4 KB), Character ROM (4 KB), and C64 KERNAL/BASIC for hardware C64 mode.
  - Register and wire components: Dual CPU slots (8502 + Z80), 128 KB dual-bank RAM, MMU 8722, VIC-IIe, VDC 8563, Dual CIA 6526, SID 6581/8580, IEC bus.
  - Implement 108-key keyboard matrix mapping (standard keys + numeric keypad, Alt, Caps Lock, 40/80 Display key, ESC, TAB, Help, Line Feed).

### Phase 34.7: C128 Integration Tests
*Goal: End-to-end testing of C128 operation.*

- [ ] **Unit and Integration Suite** (`tests/test_c128.cpp`):
  - MMU translation test: Verify RAM Bank 0/1 data isolation and Shared RAM protection.
  - Dual CPU handoff test: Verify Z80 startup and yield to 8502.
  - VDC test: Write/read registers via `$D600/$D601`, verify VDC RAM block fill/copy.
  - Mode switching test: Execute `GO 64` sequence and verify system transitions to C64 memory map.
